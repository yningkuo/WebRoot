  1. W3C Geolocation-sensor Overview

    W3C has a clear definition about how Geolocation Sensor Interface obtain the geolocation of the hosting device. Also provided with several useful code examples to introduce how to code for Geolocation Sensors.
    The Geolocation Sensor can both operate in the foreground and background. In the foreground operation, it can get a one-off geolocation update. Take Google maps for example, it just like you type in a location and search. It can also do continuous geolocation updates. Like using Google Maps for navigating to your destination while driving. For the foreground operation, it can get a one-off geolocation update like using on Ride share application or Retail special offers. It can also do continuous geolocation updates, such as tracking sports activity.
    With these powerful feactures, I think it's very good to apply Geolocation Sensor on the app for tracking the user's location and show the related advertisements.

    Rating: 5 / 5

  2. Code Labs

    Code labs provide with a detail information about how to get sensor data via a mobile app. This is very useful for building an app which utilizes sensors.
    However, the device camera, fingerprint sensor, microphone, and GPS (location) sensor all have their own APIs and are not considered part of the Android sensor framework. Therefore, they will not be included in this page.

    Rating: 5 / 5

  3. Android Developers - Sensor

    In this article, Android Developers provides with very detailed introduction to all of the sensors in the Android sensor framework. It also shares the knowledge about how to test with Android Emulator, which has an awesome interface and easy to use.
